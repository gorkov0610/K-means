## Build   
Достапни се две конфигурации на програмата. Завнисно од потребите параметрите може да се проследат како аргумент во време на извршување или како влез откако ќе започне програмот. 
За влез се користи JSON објект, во него се специфицира **датасет**, **полиња од датасетот**, **максимален број на кластери**, **број на кластери** и **центроиди**. Влезот изгледа вака:        
  
>{  
>    "dataset":"path/to/dataset",     
>    "fields":["names", "of", "fields"],      
>    "iterations":10,  
>    "numClusters":2,  
>    "centers":[{"x":0.15, "y":0.1, "z":12.2}, {"x":9.5, "y":1.7, "z":1.1}]  
>}   

На излез од програмот повторно се добива JSON објект во кој се содржани **центроидите на кластерите** и **кластерите**, двете полиња се низа од низи. Погледни го `output.json`. 
Последната верзија правилно ко искористува механизмот за конкурентност со редица од нишки, користи ја неа за најбрзо извршување.
